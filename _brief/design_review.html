②디자인 강점 (잘 된 점)

1. 반도네온 일러스트 — 시그니처 자산

호수+산+소나무+탱고 커플이 반도네온 안에 들어가는 커스텀 그림. "춘천 + 탱고"라는 정체성을 한 장으로 표현. 어디 다른 페스티벌에 없는 우리만의 마스코트. cream/dark 두 버전으로 제공되어 어느 배경에도 사용 가능.

2. 두 오케스트라 셀링포인트 부각

Hero 다음 두 번째 섹션 = 오케스트라 풀블리드 카드. "From Buenos Aires", "★ TWO LIVE ORCHESTRAS ★", aspect-ratio 4:5 큰 카드 — 스크롤 즉시 "이게 뭔지" 인지. 이게 KCCTF의 핵심 차별점이라는 걸 한 눈에 보여줌.

3. 한글 우선 + 영문 보조 일관성

모든 헤드라인이 .bigKr + .bigEn 두 줄 패턴 (Noto Serif KR 큰 헤드라인 + Playfair Italic 영문 부제). "땅고 바르도 / Tango Bardo", "여섯 명의 DJ / Six DJs". i18n 토글 없이도 한 화면에 두 언어가 우아하게 공존.

4. 타이포 시스템이 정교함

Noto Serif KR (한글 헤드라인), Noto Sans KR / Pretendard (한글 본문), Playfair Display (영문 디스플레이, italic), Inter (영문 본문), Oswald (eyebrow caps). letter-spacing 가이드(-0.04em ~ 0.32em)도 명시. 즉시 구현 가능.

5. 디자인 토큰 완비

컬러(primary/accent/neutrals/dark/semantic), 타이포 스케일(clamp 사용), 스페이싱(4px base), 라디우스, 섀도, 보더까지 CSS 변수로 정의. Tailwind config 스니펫까지 약속됨.

6. JSX 프로토타입 9개

Nav · Hero · Lineup · Lessons · MoodMakers · Tickets · Venue · MoodCollage · Footer. 그대로 React/Next.js 코드베이스에 넣어 재구성 가능. 인라인 스타일이라 변경도 쉬움.

7. 인터랙션 가이드

섹션 진입 fade+translateY(16→0, 600ms cubic-bezier), 사진 hover scale 1.03 + saturate 1.15, CTA hover 색상, Nav 스크롤 섀도까지 정의. Framer Motion 권장. 절제 톤(과하지 않게).

8. 카피 톤이 BTE급

"호숫가에서 사흘 밤. 부에노스 아이레스가 춘천에 머무는 시간." / "And so we begin — by the lake." / "여섯 번의 수업, 한 호흡으로." — 시적이지만 가볍지 않음. 한국어 카피가 영어 카피만큼이나 정성스러움.

충돌 #3 — 회차: 제3회 vs 명시 없음

⑦다음 단계 — Builder 직진 가능

받은 디자인 시스템 덕분에 Aesthetic 모드(정적 HTML 프로토타입) 단계는 스킵 가능합니다. 결정사항만 정리되면 바로 Builder 모드로 갈 수 있어요.

제 추천 진행 순서
  • Step 1 (오늘): 위 8가지 결정사항 중 1·2·3은 즉시 결정 가능 (제 추천대로 가시거나 다른 선택). 4~7은 운영팀 확인 필요.
  • Step 2: Wordsmith 모드로 운영기준의 정확한 데이터(가격/일정/라인업/장소)를 디자인 카피 톤에 맞춰 한/영 동시 작성. 디자인의 자리표시자(placeholder) 텍스트를 진짜 데이터로 교체할 준비.
  • Step 3: Builder 모드로 Next.js 14 (App Router) + Tailwind + next-intl로 새 사이트 구축. 받은 JSX를 바탕으로 재구성.
  • Step 4: Critic 모드로 모바일·접근성·운영기준 일치 검수.
  • Step 5: 임시 도메인 배포 → 운영진 확인 → 도메인 스왑 또는 기존 사이트 교체.

  • (A) Wordsmith — Hero 카피 한/영 후보 5개 (디자인 톤 + 운영기준 사실 결합)
