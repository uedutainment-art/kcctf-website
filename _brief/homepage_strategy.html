📌한눈 요약

결론: 현재 kcctf.org는 "공지 페이지" 수준. 벤치마크인 Belgrade Tango Encuentro는 "경험을 미리 살게 만드는" 수준. 갭이 크지만 콘텐츠(운영기준 MD)는 거의 다 있어서, 바이브 코딩으로 1~2주 안에 충분히 BTE급 사이트로 끌어올릴 수 있음.

🟢 잘되어 있는 것

  • 운영 데이터 정리됨 (운영기준.md = 콘텐츠 90%)
  • 한/영 토글 구조 존재 (Next.js + Firebase)
  • 고급스러운 골드/다크 디자인 시스템 확립 (대시보드/홍보동선/제작물 전부 동일 톤)
  • 오케스트라 2팀 + DJ 6명 라인업 강력
  • 홍보 동선 활발 (해외 5개국 + 국내 다수)

🔴 시급한 갭

  • 등록 시스템 부재 ("Coming soon" 상태)
  • 가격/패키지 정보 없음 (얼리버드 19만원, 풀팩+호텔 등)
  • 호텔/여행 정보 없음 (벨라스테이 확정인데)
  • 댄서팀 정보 누락 (London Hong & Sol 등 4팀)
  • 요일 표기 오류 (10/3 토 → "Friday session"으로 표기)
  • M2 시간 불일치 (홈 22:00 vs 계획 21:00)
  • 스토리텔링/감성 카피 부재 (BTE 대비)
  • 갤러리/지난 회차 / FAQ / 여행 가이드 없음

⑤AI 팀 구성안 — 바이브 코딩 페르소나

이 Cowork 세션 안에서 운영 가능한 6개 AI 페르소나. 작업 시 "지금은 [역할명]으로"라고 호출하면 그 페르소나로 응답.

P1 의사결정 / 비전
🎬 크리에이티브 디렉터 — "Director"
책임: 전체 비전, 브랜드 톤, 핵심 메시지, 컬러/타이포 결정. 인풋: 운영기준, 행사 본질, 타겟 정서. 아웃풋: 브랜드 가이드, 카피 톤, 모드보드.
  • "Tango Bardo & Misteriosa 특별 콘서트"가 우리만의 무엇인지 한 문장으로 정의
  • 해외 vs 국내 참가자 메시지 우선순위 결정
  • 최종 디자인/카피 검수 ("이게 우리 KCCTF 맞아?")

→ 포올 + Director(나)가 페어로 결정.

P2 콘텐츠 / 카피
✍️ 카피라이터 — "Wordsmith"
책임: 모든 텍스트의 톤과 디테일. 한국어/영어 둘 다. 인풋: Director의 톤 가이드, 운영기준 데이터. 아웃풋: Hero 카피, About 내러티브, 밀롱가 분위기 묘사, FAQ, CTA 버튼 문구.
  • BTE 수준의 감성 카피 ("And so we start..." 같은 톤)
  • 한국어 우선 + 영어 동시 작성 (해외 참가자 비중 고려)
  • Tango Bardo / Misteriosa 같은 라인업의 브랜드성을 살린 소개문
P3 디자인 / UX
🎨 UI/UX 디자이너 — "Aesthetic"
책임: 시각 디자인, 레이아웃, 컴포넌트 패턴, 반응형. 인풋: 기존 디자인 시스템(골드 #C9A84C / 다크 #1A1410), BTE 레이아웃 참고. 아웃풋: HTML/CSS 시안, 컴포넌트 라이브러리.
  • 대시보드/홍보동선 톤 그대로 가져가서 일관성 유지 (이미 너무 좋음)
  • Hero 이미지 캐러셀 / 마에스트로 카드 / 밀롱가 그루핑 카드
  • 모바일 우선 (해외 참가자는 폰으로 봄)
P4 구현
💻 프론트엔드 개발자 — "Builder"
책임: 실제 코드 작성, 배포. 인풋: Aesthetic의 시안, Wordsmith의 카피. 아웃풋: 작동하는 사이트.
  • 옵션 A: 현재 Next.js 기반 그대로 + 페이지 추가 (개발자 협업)
  • 옵션 B: 정적 HTML로 별도 제작 후 통째로 교체 (빠름)
  • 옵션 C: Astro/Vite로 가볍게 새로 (중간)
  • 한/영 i18n 구조 유지
P5 검수 / 품질
🔍 QA 리뷰어 — "Critic"
책임: 오타, 사실 오류, 깨진 링크, 모바일 깨짐, 접근성 검수. 인풋: 완성된 페이지. 아웃풋: 수정 리스트.
  • 현재 사이트의 "Friday session" 같은 오류 더 있는지 전수 검사
  • 한국어/영어 오타 점검
  • 운영기준 데이터와 실제 사이트 일치 확인 (날짜/시간/가격)
  • 모바일 320px / 태블릿 / 데스크톱 깨짐 점검
P6 운영 / 외부 연결
📋 프로젝트 매니저 — "Coordinator"
책임: 외부 의존성(흰곰/조앤/존) 추적, 일정, 우선순위. 인풋: 운영기준 §11(확인 대기 목록). 아웃풋: 무엇을 누가 언제까지 결정해야 사이트가 진행되는지 매트릭스.
  • 흰곰 결정 = 가격/패키지/등록방식 → Register 페이지 차단 요소
  • 조앤 결정 = 프로그램 확정/M2 시간 → Program 페이지 차단 요소
  • 존 결정 = 댄서팀 확정 → Artists 페이지 차단 요소
  • 차단 요소가 있어도 "Coming soon" 플레이스홀더로 우회 진행 가능
💡 운영 팁: 한 번에 한 명만 호출. 예) "디렉터 모드로 Hero 메시지 후보 5개 뽑아줘", "카피라이터로 About 한국어/영어 두 가지 톤(서정적 vs 단정적)으로 써줘", "QA로 운영기준 §3 vs 현재 kcctf.org 불일치 전수 점검". 역할 명확하면 결과물 품질이 훨씬 올라감.

Director가 주도. 결과물: 무드보드 + 핵심 메시지 + 톤 가이드 1장.
  • "Tango Bardo + Misteriosa = 한국에 처음 오는 보석"이라는 포지셔닝 결정
  • 해외 vs 국내 메시지 비중 (예: 70:30)
  • BTE에서 가져올 것 / KCCTF 고유로 만들 것 구분
2
Phase 2 — 콘텐츠 + 시안 동시 진행 (3~4일)
