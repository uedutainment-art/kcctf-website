①현재 사이트 분석

외부에서 확인 가능한 정보로 분석한 결과입니다 (코드 접근 권한이 있으면 더 정확하게 보충 가능).

항목 현재 상태 의미
도메인 kcctf.org 커스텀 도메인 — DNS는 누가 관리하는지 확인 필요 (이종화 이사장?)
실제 호스팅 kcctf-5047d.web.app Firebase Hosting. kcctf-5047d는 Firebase 프로젝트 ID.
Canonical URL kcctf-5047d.web.app/events/chuncheon-citf-2026 multi-event 구조 — KCCTF는 더 큰 시스템의 한 이벤트 페이지일 가능성.
"events/" 경로는 다른 이벤트도 호스팅 가능하게 설계됨.
프레임워크 Next.js (이미지 URL이 /_next/image?url=... 형태) 현재 사이트도 Next.js. 우리도 같은 스택으로 가서 호환성 좋음.
i18n ?lang=ko / ?lang=en 쿼리 파라미터 방식. 우리는 /ko / /en 라우트 방식으로 가는 게 SEO 더 좋음.
이미지 자산 위치 /images/events/chuncheon-citf-2026/* 이벤트별로 폴더 분리. 우리 새 사이트는 단일 이벤트라 /public 직접.

이전 작업 방식 추정

추정: 누군가(이종화 이사장 또는 협력 개발자)가 만든 multi-event Firebase 프로젝트에 KCCTF 페이지를 한 이벤트로 추가. kcctf.org 도메인을 그 프로젝트로 매핑.

확인 필요:
  • Firebase 프로젝트 kcctf-5047d의 소유자/관리자가 누구인지
  • 도메인 kcctf.org 등록자(registrar)와 DNS 관리자가 누구인지
  • 이전에 다른 회차(2024, 2025)는 같은 시스템 안에서 어떻게 보존되었는지

→ 이종화 이사장에게 확인. 운영기준에는 이 정보 없음.

단계별 액션

1
Cowork에서 패키지 만들기 (오늘)
자산(design 폴더) + 명세서(CLAUDE.md) + 운영기준 + 와이어프레임 → 단일 폴더로 zip. 노트북에 옮길 준비.
2
노트북에 패키지 풀기
~/projects/kcctf-2026 같은 작업 폴더 만들고 zip 풀기. 자산이 ./_brief/ 또는 ./_design/ 하위에 정리됨.
3
노트북에서 Claude Code 시작
터미널에서 cd ~/projects/kcctf-2026 && claude → 첫 메시지에 "CLAUDE.md 읽고 Phase 4 (스캐폴딩) 시작해줘"
4
Claude Code가 자동으로 빌드
CLAUDE.md 따라 Next.js 프로젝트 생성 → 디자인 토큰 → 컴포넌트 → 데이터 채우기. 중간중간 멈추고 확인 요청.
5
GitHub repo 만들고 Vercel 연결
Claude Code가 git init → GitHub에 푸시 → Vercel 가져오기 → kcctf-2026.vercel.app 자동 생성.
6
스테이징 URL 운영팀 공유
존·흰곰·조앤·지이에게 카톡으로 URL + 체크리스트 공유. 사실관계 확인 + 디자인 피드백 받음.
7
수정 반복 (이 단계 길어질 수 있음)
운영팀 피드백 → Claude Code로 수정 → git push → 자동 재배포. 같은 URL에서 갱신.
8
컨펌 후 도메인 swap
이종화 이사장에게 DNS 변경 요청 (Vercel이 알려준 A/CNAME 값). 5~30분 후 kcctf.org가 새 사이트로.
9
이전 사이트 처리
옵션: archive.kcctf.org에 보존 / 그대로 Firebase에 두고 사용 안 함 / 완전 삭제. archive 권장 (이전 회차 자료 보존).
