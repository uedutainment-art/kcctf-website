②새 폴더는 언제·어떻게 만들까?

5/13 (화) 이후 만드세요. 그 전까지는 안 만들어도 됩니다.

5/13에 만들 때 따라할 단계입니다 (Mac 기준):

1
터미널 열기

Mac에서 Spotlight (cmd + space) → "터미널" 입력 → 엔터.

까만 창 열림. 여기에 명령어를 한 줄씩 칠 거예요.

2
현재 폴더가 어디 있는지 확인

현재 작업 중인 폴더가 어디 있는지부터 확인. 보통 이런 위치 중 하나일 거예요:

~/projects/kcctf
~/dev/kcctf
~/kcctf
~/Documents/kcctf
~는 "내 사용자 홈 폴더"라는 뜻 (예: /Users/uedutainment).

모르겠으면 Finder에서 폴더를 찾아서 우클릭 → "옵션" 키 누르고 → "경로 이름 복사" 하면 정확한 경로가 복사됩니다.

3
현재 폴더의 옆에 새 폴더 만들기

예를 들어 현재 폴더가 ~/projects/kcctf에 있다면:

cd ~/projects
cd = "이 폴더로 이동" 명령. 이 명령으로 projects 폴더 안으로 들어감. 
mkdir kcctf-website
mkdir = "make directory" — 새 폴더 만들기. kcctf-website라는 새 폴더 생성. 
cd kcctf-website
새로 만든 kcctf-website 폴더 안으로 들어감.

이제 폴더 구조가 이렇게 됩니다:

~/projects/
├── kcctf/             ← 기존 폴더 (그대로)
└── kcctf-website/     ← 새 폴더 (지금 만든 것, 비어있음)
4
패키지 zip 풀기

제가 만들어드릴 kcctf-2026.zip을 다운로드하셨다면 (보통 ~/Downloads/에 있음):

unzip ~/Downloads/kcctf-2026.zip -d .
unzip = zip 파일 풀기. -d . = "현재 폴더(.)에 풀어라".

이제 kcctf-website 폴더 안에 이런 게 들어옵니다:

kcctf-website/
├── CLAUDE.md          ← Claude Code가 읽을 명세서
├── _brief/            ← 운영기준, 디자인 리뷰 등 참고 자료
└── _design/           ← 디자인 시스템 (이미지·로고 등)
5
Claude Code 시작

같은 터미널에서 (이미 kcctf-website 폴더 안에 있는 상태):

claude
Claude Code가 시작됩니다. 이제 한국어로 명령을 칠 수 있어요.

첫 메시지로 이렇게 보내세요:

지금 이 폴더(현재 working directory)에서만 작업해.
부모 폴더, 형제 폴더는 절대 읽지도 수정하지도 마.

CLAUDE.md를 먼저 읽고, _brief/ 안의 운영기준.md / 디자인_리뷰 / 새_Hero_시안도 훑어봐.
컨텍스트 잡히면 Phase 4 시작 — Next.js 14 + Tailwind + next-intl 스캐폴딩.
첫 줄이 핵심: "이 폴더에서만 작업해". 이러면 Claude Code가 옆에 있는 kcctf(현재 폴더)는 절대 안 건드림.
6
Claude Code가 알아서 빌드

이후로는 Claude Code가 명세서 따라 자동으로 빌드합니다. 중간중간 멈추고 물어봐요. 답해주면 됩니다.

모르는 게 나오면 그냥 "잘 모르겠어, 추천해줘" 라고 답해도 됩니다.

🔍헷갈릴 때 체크리스트

"내가 지금 어느 폴더에 있는지" 확인하는 명령: 
pwd
pwd = "print working directory" — 현재 위치 출력. 결과가 /Users/uedutainment/projects/kcctf-website 라면 새 폴더 안. 만약 /Users/uedutainment/projects/kcctf라면 현재 폴더 안 — 잘못 들어간 것!
"이 폴더 안에 뭐가 있는지" 확인하는 명령: 
ls
ls = "list" — 폴더 내용 보기. 새 폴더에서 치면 CLAUDE.md _brief _design이 보여야 정상.
잘못 만들었거나 다시 시작하고 싶으면: 
cd ~/projects
rm -rf kcctf-website
rm -rf = 폴더 통째로 삭제. 주의: 이건 kcctf-website 폴더만 삭제. 절대 옆에 있는 kcctf 폴더에는 영향 없음. 하지만 명령 칠 때 폴더 이름은 정확히!

❓자주 묻는 질문 (FAQ)

Q. 두 폴더가 있으면 어느 걸 작업해야 할지 헷갈릴 것 같아요.
A. 폴더 이름으로 구분하세요. kcctf-platform = 운영, kcctf-website = 홈페이지. 터미널 명령 pwd로 항상 어디 있는지 확인 가능. Claude Code도 시작할 때 어느 폴더에서 시작했는지에 따라 작업 범위가 자동으로 정해져요.
Q. 두 폴더의 git이 충돌하지 않을까요?
A. 안 합니다. 별도 폴더 = 별도 git repo. 서로 완전 독립. git push도 따로, GitHub repo도 따로.
Q. 같은 노트북에서 두 개를 동시에 작업해도 되나요?
A. 됩니다. 터미널 창을 두 개 열어서 한 창은 platform, 한 창은 website. 또는 Claude Code도 두 개 동시 실행 가능. 하지만 5/12 전까지는 platform 하나만 — 동시 작업 부담 줄이기 위해.
Q. 새 폴더를 평소 폴더(예: ~/projects/) 말고 바탕화면이나 다른 곳에 만들어도 되나요?
A. 됩니다. 단 한국어 폴더명·공백·특수문자 들어간 경로(예: 바탕화면 = ~/Desktop/인데 한국어 OS면 다를 수 있음)는 가끔 명령어가 꼬일 수 있으니 영어로 된 경로 추천. ~/projects/가 가장 안전.
Q. Claude Code가 자꾸 다른 폴더 보려고 하면요?
A. "현재 폴더에서만 작업해. 부모/형제 폴더 절대 안 읽어도 됨"이라고 한 번 더 말씀하시면 됩니다. 또는 CLAUDE.md에 명시되어 있어서 자동으로 그렇게 동작합니다.
Q. 5/12 오픈 후에도 옛 사이트(현재 kcctf.org)는 그대로인가요?
A. 네 그대로. 새 홈페이지가 완성되고 운영팀이 OK 할 때까지 옛 사이트는 살아있음. 도메인 변경은 마지막에 한 번만.
Q. 새 폴더 만들기 전에 미리 준비할 게 있나요?
A. 거의 없어요. 제가 zip 만들어드리면 다운로드만 미리 해두시면 됩니다 (5/13에 풀어서 시작). GitHub 계정 + Vercel 계정도 있으면 좋음 (없으면 5/13에 같이 만들면 됨, 5분 작업).
Q. Claude Code가 처음이라 잘 못 다룰까봐 걱정돼요.
A. 평소 한국어로 말하듯이 명령하면 됩니다. "Hero 만들어줘", "이 부분 수정해줘", "빨간색을 더 진하게" 같은 식. Claude Code가 영어 명령으로 알아서 변환해서 작업해줍니다. 모르겠으면 "잘 모르겠어, 추천해줘"라고 해도 OK.
