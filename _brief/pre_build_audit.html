옵션 1 — 단순 (당초 계획)
기존 자료만 묶기 · 30분 소요
운영/ 폴더 보고서 + design/ 자산 + 간단한 CLAUDE.md.
문제: Critical 5건 누락 → 빌드 도중 막힘.
옵션 2 — 보강 (추천)
Critical 5건 + Important 6건 보강 · 2~3시간
옵션 1 + 한/영 카피 미리 작성 + 법적 페이지 텍스트 + Favicon/OG 변환 + 운영자 매뉴얼 + 환경 변수 + SEO + 학습 가이드.
장점: Claude Code가 처음부터 끝까지 막힘 없이 빌드 가능. 사용자도 헷갈림 적음.
옵션 3 — 풀 보강
Nice-to-have까지 모두 · 1일+
옵션 2 + DJ 약력 + Past Editions + Visiting Chuncheon + 추가 위젯.
단점: 미정 항목 많아서 placeholder 폭증, 의미 없음.
강력 추천 — 옵션 2. Critical은 반드시, Important는 거의 다 포함, Nice-to-have는 패스. 저는 2~3시간 안에 보강 패키지 만들 수 있습니다. 단, 법적 페이지 텍스트(이용약관·개인정보·환불)만은 포올님이 현재 사이트에서 복사해 주셔야 함 — 이건 운영팀이 작성한 콘텐츠라 제가 만들면 안 됨.

⑪다음 액션

포올님 답변 부탁드릴 4가지
  1. 옵션 2(보강)로 진행 OK인지? (3시간 작업)
  2. 법적 페이지 텍스트 — 현재 kcctf.org/terms, /privacy, /refund 들어가서 텍스트 복사해서 보내주실 수 있는지? (이번 주 안에)
  3. Analytics — Vercel Analytics(무료, 자동) + GA4(트래픽 분석용)? 둘 다 OK?
  4. Open Graph 이미지 — 받은 포스터(reference-poster-2026.jpeg)를 1200×630으로 크롭해서 OG로? 또는 별도 디자인 필요?

병렬로 운영팀에 던질 카톡 (이번 주 안)

  • 이종화: 도메인·Firebase·이전 회차 자료 (5가지 — 이미 초안 있음)
  • 흰곰: 콘서트 장소·등록 방식·얼리버드 마감일·호텔 가격 (4가지)
  • 조앤: 회차·워크샵 운영·M2 시간·라인업 확정 (4가지)
  • 존: 댄서팀 사진·4번째 댄서 확정 (2가지)

→ 답이 늦어도 빌드는 placeholder로 진행. 답 떨어지는 대로 환경 변수만 변경.
